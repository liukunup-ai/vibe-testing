### 关键交互流程

**流程一：创建任务并执行**

```
1. 用户进入 [任务列表] 页面
2. 点击 [+创建任务]
3. 进入 [创建任务] 向导
   - 步骤1: 填写基本信息（任务名称、项目、类型、触发方式）
   - 步骤2: 选择执行范围（用例/套件/标签）
   - 步骤3: 配置环境变量
   - 步骤4: 选择目标设备
   - 步骤5: 设置高级选项（重试、通知等）
4. 点击 [立即执行] 或 [保存]
5. 跳转到 [执行监控] 页面
6. 实时显示执行进度和日志
7. 执行完成后生成报告
8. 用户可查看报告或导出
```

**流程二：执行过程中发现问题并处理**

```
1. 用户在 [执行监控] 页面看到设备执行失败
2. 点击 [诊断] 按钮
3. 系统展示：
   - 失败用例的详细错误信息
   - 截图和视频
   - 相关日志片段
   - 智能诊断建议
4. 用户选择：
   - [重试] 按钮：重新执行该用例
   - [跳过] 按钮：跳过该用例继续执行
   - [终止] 按钮：停止整个任务
   - [标记已知问题]：添加到知识库
5. 处理完成后继续监控
```
